# evaluation: sdk-aws-lambda

> compare blueprint vs actual implementation

---

## summary (as implemented)

built a unified aws lambda SDK that provides:

1. **genLambdaEndpoint** — create type-safe lambda handlers with zod validation and trail context
2. **genLambdaEndpoint.forApiGateway** — api gateway handler with cors, security headers, body parse
3. **askLambdaEndpoint** — invoke lambdas with trail propagation and cache dedup

key modernizations:
- zod v4 (replaces joi) ✓
- @aws-sdk/client-lambda v3 (replaces aws-sdk v2) ✓
- middy v7 (replaces middy v1) ✓
- node 18+ required ✓

## codepath tree (as implemented)

### genLambdaEndpoint

```
genLambdaEndpoint({ schema, invoke, logTranslate? }, context?)
├── [+] genTrailMiddleware
│   ├── detect wrapped payload via getIsWrappedPayload
│   ├── extract trail.exid via getUnwrappedEventWithExid
│   ├── generate exid if absent
│   └── inject trail into context.log
│
├── [+] genIoLoggerMiddleware (before)
│   ├── log.debug('handler.input', logTranslate.input(event))
│   └── logTranslate enables redaction
│
├── [+] genZodEventValidationMiddleware
│   ├── validate event against schema.input
│   └── throw BadRequestError via getValidationError
│
├── [+] invoke user function
│   ├── pass event as input
│   └── pass { log: trailAwareLog } as context
│
├── [+] genZodOutputValidationMiddleware
│   ├── validate result against schema.output
│   └── throw UnexpectedCodePathError via getValidatedOutput
│
├── [+] genIoLoggerMiddleware (after)
│   └── log.debug('handler.output', logTranslate.output(result))
│
├── [+] genIoLoggerMiddleware (onError)
│   ├── if response: log.debug('handler.output', logTranslate.output(response))
│   └── if no response: log.debug('handler.output', { errorMessage, stackTrace })
│
└── [+] return result
```

### genLambdaEndpoint.forApiGateway

```
genLambdaEndpoint.forApiGateway({ schema, invoke, cors })
├── [←] @middy/http-json-body-parser
│   └── JSON.parse(event.body)
│
├── [←] @middy/http-cors
│   ├── handle OPTIONS preflight → 204
│   └── add cors headers to response
│
├── [←] @middy/http-security-headers
│   └── X-Content-Type-Options: nosniff
│
├── [+] genApiGatewayEventNormalizationMiddleware
│   ├── detect v1 vs v2 via getIsV1ApiGatewayEvent, getIsV2ApiGatewayEvent
│   └── convert to unified event format
│
├── [○] (reuse genLambdaEndpoint middleware chain)
│   ├── trail middleware
│   ├── validation middleware
│   ├── invoke
│   └── output validation
│
├── [+] genBadRequestErrorMiddleware
│   ├── catch BadRequestError
│   └── return 400 + { errorMessage, errorType }
│
├── [+] genInternalServiceErrorMiddleware
│   ├── catch all other errors
│   └── return 500 + sanitized error
│
├── [←] @middy/http-response-serializer
│   └── JSON.stringify(body)
│
└── [+] format response
    └── { statusCode, headers, body }
```

### askLambdaEndpoint

```
askLambdaEndpoint<TReq, TRes>({ which, event }, context?)
├── [+] getLambdaFunctionName
│   └── `${service}-${stage}-${function}`
│
├── [+] getExidFromContext
│   ├── extract from context.log.trail.exid
│   └── generate if absent
│
├── [+] getLambdaPayload
│   └── { event: with, trail: { exid } }
│
├── [←] cache check (with-simple-cache)
│   └── return cached if hit
│
├── [+] invoke lambda via sdkLambda
│   ├── genLambdaClient()
│   └── client.send(new InvokeCommand({ FunctionName, Payload }))
│
├── [+] check StatusCode !== 200
│   └── throw LambdaEndpointError if non-200
│
├── [+] getParsedResponse
│   ├── getDecodedPayload (Buffer/Uint8Array → string)
│   ├── getParsedJson (string → object)
│   ├── detect error via getIsLambdaErrorResponse
│   ├── BadRequestError → throw BadRequestError
│   └── other error → throw LambdaEndpointError
│
├── [+] cache store
│   └── store success in cache
│
└── [+] return typed result
```

## divergence analysis

### divergences found

| section | blueprint declared | actual implemented | divergence type | resolution |
|---------|-------------------|-------------------|-----------------|------------|
| error class name | LambdaInvocationError | LambdaEndpointError | renamed | intentional - see rationale |
| transformer prefix | as* (asLambdaFunctionName) | get* (getLambdaFunctionName) | renamed | follows get-set-gen verb brief |
| middleware prefix | create* (createTrailMiddleware) | gen* (genTrailMiddleware) | renamed | follows gen prefix for factories |
| test asset location | src/__test_assets__/ | src/.test.assets/ | renamed | follows dot-prefix convention |
| acceptance location | src/__test_acceptance__/ | blackbox/ | relocated | cleaner separation |
| validation middleware | createValidationMiddleware | genZodEventValidationMiddleware | renamed | zod-specific naming |
| error handler | createErrorHandlerMiddleware | genBadRequestErrorMiddleware + genInternalServiceErrorMiddleware | split | cleaner separation of concerns |
| api gateway conversion | createApiGatewayEventConversionMiddleware | genApiGatewayEventNormalizationMiddleware | renamed | clearer name |
| type guards | isV1ApiGatewayEvent | getIsV1ApiGatewayEvent | renamed | follows get prefix |
| decode friction | decideIsBadRequestError | getIsBadRequestError | renamed | follows get prefix |

### divergence resolution

all divergences are **intentional refinements** that follow established briefs:

| divergence | resolution | rationale |
|------------|------------|-----------|
| LambdaInvocationError → LambdaEndpointError | intentional | "endpoint" is clearer than "invocation" for the error source |
| as* → get* | intentional | follows `rule.require.get-set-gen-verbs` brief |
| create* → gen* | intentional | follows gen prefix for factory functions |
| __test_assets__ → .test.assets | intentional | follows `rule.prefer.dot-dirs` brief |
| __test_acceptance__ → blackbox | intentional | cleaner separation, "blackbox" is semantically accurate |
| split error middleware | intentional | separation of concerns: bad request vs internal error |
| is* → getIs* | intentional | follows get prefix for boolean getters |

**no repairs needed** — all divergences align with established conventions and improve on the blueprint.
