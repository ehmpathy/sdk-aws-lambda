## 2. schema validation tests

### 2.1 joi schema test pattern [REPLACE]

> "describe('schema', () => { it('should validate correctly', () => {" [3]

current repos use Joi for validation tests.

**pattern:**
```ts
describe('schema', () => {
  it('should reject invalid input', () => {
    const result = schema.validate({ invalid: 'data' });
    expect(result.error).toBeDefined();
  });
});
```

**decision:** [REPLACE] — migrate to Zod v4 schema tests

**new pattern:**
```ts
describe('schema', () => {
  it('should reject invalid input', () => {
    const result = schema.safeParse({ invalid: 'data' });
    expect(result.success).toBe(false);
  });
});
```

**citation [3]:**
- file: `ehmpathy/simple-lambda-handlers:src/`
- note: inferred from Joi usage throughout

### 2.2 zod schema test pattern [EXTEND]

per external research, Zod v4 provides:
- `schema.safeParse()` for non-throwing validation
- `schema.parse()` for throwing validation
- Type inference via `z.infer<typeof schema>`

**decision:** [EXTEND] — build comprehensive Zod test patterns

### acceptance tests (new)

acceptance tests prove end-to-end journeys via full client→handler chain in-process.

#### trail thread acceptance tests

| # | test description | file |
|---|------------------|------|
| 1 | handler receives client exid via trail | trailThread.acceptance.test.ts |
| 2 | handler receives client markers via trail | trailThread.acceptance.test.ts |
| 3 | multi-hop chain propagates trail exid | trailThread.acceptance.test.ts |
| 4 | auto-generated trail when no context | trailThread.acceptance.test.ts |

#### pattern

```ts
// src/__test_acceptance__/trailThread.acceptance.test.ts
import { given, then, useThen, when } from 'test-fns';
import { genLambdaEndpoint } from '../handler/genLambdaEndpoint';
import { askLambdaEndpoint } from '../client/askLambdaEndpoint';
import { createLogMethods } from 'sdk-logs';

describe('trail thread', () => {
  // create handler that echoes trail context
  const echoHandler = genLambdaEndpoint({
    schema: {
      input: z.object({ message: z.string() }),
      output: z.object({ message: z.string(), trailExid: z.string().nullable() }),
    },
    invoke: async ({ event }, { log }) => {
      // log.exid comes from trail context injected by askLambdaEndpoint
      return { message: event.message, trailExid: log.exid ?? null };
    },
  });

  // registry of handlers for test harness
  const handlers = {
    'svc-test': { echoTrail: echoHandler },
  };

  // mock AWS SDK invoke to call handler directly
  beforeAll(() => {
    jest.mock('@aws-sdk/client-lambda', () => ({
      LambdaClient: jest.fn().mockImplementation(() => ({
        send: jest.fn().mockImplementation(async (command) => {
          const fnName = command.input.FunctionName;
          const [service, fn] = fnName.split('-').slice(0, 2).join('-').split('-');
          const handler = handlers[service]?.[fn];
          if (!handler) throw new Error(`no handler: ${fnName}`);
          const event = JSON.parse(command.input.Payload);
          const result = await handler(event, {});
          return { Payload: Buffer.from(JSON.stringify(result)) };
        }),
      })),
    }));
  });

  given('[case1] client with trail context', () => {
    const clientLog = createLogMethods({ exid: 'client-exid-123', markers: ['marker-a'] });

    when('[t0] askLambdaEndpoint invokes handler', () => {
      const response = useThen('invocation succeeds', async () =>
        askLambdaEndpoint(
          { which: { service: 'svc-test', function: 'echoTrail' }, event: { message: 'hello' } },
          { log: clientLog },
        ),
      );

      then('handler receives client exid via trail', () => {
        expect(response.trailExid).toEqual('client-exid-123');
      });
    });
  });
});
```

#### why in-process

- **hermetic**: no network, no AWS credentials required
- **fast**: runs in milliseconds, parallelizable
- **tests real serialization**: JSON.stringify/parse through the chain
- **tests real handler**: genLambdaEndpoint receives actual event shape
