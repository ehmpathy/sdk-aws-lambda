# blocker.1: optional attributes in internal contract input

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/pitofsuccess.procedures/rule.forbid.undefined-inputs.md.min

**locations**:
- src/domain.operations/askLambdaEndpoint/getLogForExidExtraction.ts:1

The function `getLogForExidExtraction` accepts an input with optional properties (`trail?` and `exid?`). According to the rule, optional attributes in internal contract inputs are blockers because they hide missing values and allow undefined to propagate. Use explicit `null` instead of optional/undefined for internal contracts.

The input type is:
```ts
{ log: { trail?: { exid?: string | null } } | null | undefined }
```
Should be changed to require `trail` as `{ exid: string | null } | null` and accept `null` for the whole log parameter, without optional fields.

**snippet**:
```ts
export const getLogForExidExtraction = (input: {
  log: { trail?: { exid?: string | null } } | null | undefined;
}): { trail: { exid: string | null } | null } | null => {
  if (!input.log) return null;
  if (!input.log.trail) return { trail: null };
  return { trail: { exid: input.log.trail.exid ?? null } };
};
```
