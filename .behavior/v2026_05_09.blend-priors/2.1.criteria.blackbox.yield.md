# blackbox criteria: sdk-aws-lambda

> experience boundaries — what users see, do, and receive

---

# usecase.1 = define standard lambda endpoint

## 1.1 valid input flow

given(a handler with zod input/output schemas)
  when(handler receives valid input)
    then(input is validated against schema)
    then(invoke function is called with validated event)
    then(output is validated against schema)
    then(response is returned to caller)
      sothat(caller receives typed, validated data)

## 1.2 invalid input rejection

given(a handler with zod input schema)
  when(handler receives input that fails validation)
    then(invoke function is NOT called)
    then(validation error is returned)
    then(error includes which fields failed)
      sothat(caller knows exactly what to fix)

## 1.3 invalid output detection

given(a handler with zod output schema)
  when(invoke returns output that fails validation)
    then(error is thrown)
    then(error is logged with context)
      sothat(developers catch contract violations in logs)

## 1.4 invoke error wrap

given(a handler)
  when(invoke function throws an error)
    then(error is caught and wrapped)
    then(error is logged with trail context)
    then(wrapped error is returned to caller)
      sothat(errors are observable and traceable)

## 1.5 trail context auto-generation

given(a handler)
  when(handler is invoked without trail in event)
    then(context.log is auto-generated with new trail.exid)
    then(invoke receives context with trail-aware log)
      sothat(every request is traceable even if caller forgot trail)

## 1.6 trail context propagation

given(a handler)
  when(handler is invoked with trail.exid in event)
    then(context.log inherits the caller's trail.exid)
    then(logs include the propagated exid)
      sothat(requests are traceable across service boundaries)

# usecase.2 = define api gateway endpoint

## 2.1 http request success

given(an api gateway handler with schema and cors)
  when(http request arrives with valid json body)
    then(body is parsed from request)
    then(input is validated against schema)
    then(invoke function is called)
    then(http 200 response is returned with json body)
      sothat(api consumers receive structured responses)

## 2.2 http request validation error

given(an api gateway handler with schema)
  when(http request arrives with invalid json body)
    then(http 400 response is returned)
    then(response body includes validation errors)
      sothat(api consumers know what to fix)

## 2.3 cors headers for allowed origin

given(an api gateway handler with cors origins configured)
  when(request arrives from allowed origin)
    then(response includes Access-Control-Allow-Origin header)
    then(response includes Access-Control-Allow-Credentials if configured)
      sothat(browser requests from allowed origins succeed)

## 2.4 cors preflight

given(an api gateway handler with cors configured)
  when(OPTIONS request arrives)
    then(http 204 response is returned)
    then(cors headers are included)
      sothat(browser preflight checks pass)

## 2.5 security headers

given(an api gateway handler)
  when(any request is processed)
    then(response includes security headers)
      sothat(responses follow security best practices)

## 2.6 invoke error to http error

given(an api gateway handler)
  when(invoke function throws an error)
    then(http 500 response is returned)
    then(error is logged with trail context)
    then(response body is sanitized (no stack traces))
      sothat(internal errors don't leak to api consumers)

# usecase.3 = invoke lambda from client

## 3.1 successful invocation

given(a target service and function)
  when(askLambdaEndpoint is called with payload and context)
    then(lambda is invoked with payload)
    then(trail.exid is extracted from context.log)
    then(trail is injected into the lambda call)
    then(response is parsed and returned)
      sothat(caller receives typed response with trail continuity)

## 3.2 invocation error

given(a target service and function)
  when(lambda returns an error)
    then(LambdaInvocationError is thrown)
    then(error includes original error message)
    then(error includes target service/function)
      sothat(caller knows what failed and where)

## 3.3 stage derivation

given(a target service and function)
  when(askLambdaEndpoint is called with context)
    then(stage is derived from context.env.STAGE)
    then(lambda function name is constructed with stage)
      sothat(caller doesn't need to pass stage manually)

## 3.4 cache hit

given(a target service, function, and payload)
  when(askLambdaEndpoint is called with same args twice)
    then(second call returns cached response)
    then(lambda is only invoked once)
      sothat(duplicate calls are deduplicated)

## 3.5 cache miss

given(a target service, function, and different payloads)
  when(askLambdaEndpoint is called with each payload)
    then(lambda is invoked for each unique payload)
      sothat(different requests get fresh responses)

## 3.6 context auto-generation

given(a target service and function)
  when(askLambdaEndpoint is called without context)
    then(context is auto-generated with new trail.exid)
    then(lambda call includes the generated trail)
      sothat(every call is traceable even without explicit context)

# usecase.4 = type safety

## 4.1 typed request

given(askLambdaEndpoint with generic type parameters)
  when(caller passes payload)
    then(typescript enforces payload matches request type)
      sothat(type mismatches are caught at compile time)

## 4.2 typed response

given(askLambdaEndpoint with generic type parameters)
  when(response is returned)
    then(response is typed as the response type parameter)
      sothat(callers get autocomplete and type checks)

# usecase.5 = error classification

## 5.1 validation error vs invocation error

given(a handler or client call)
  when(an error occurs)
    then(error type distinguishes validation vs invocation vs unexpected)
      sothat(callers can handle different error types appropriately)

## 5.2 error observability

given(any error in handler or client)
  when(error is thrown)
    then(error is logged with trail.exid)
    then(error includes relevant context (input, target, etc))
      sothat(errors can be traced and diagnosed in logs)
