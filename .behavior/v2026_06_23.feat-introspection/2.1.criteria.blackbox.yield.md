# blackbox criteria: introspection for lambda endpoints

## usecase.1 = contract discovery for one endpoint

```
given caller in prep environment
  when getOneLambdaContract called with valid service and function
    then returns LambdaEndpointSchema with input and output JSON schemas
      sothat caller knows what the endpoint expects and returns

  when getOneLambdaContract called with nonexistent function
    then fails with clear error that names the absent function
      sothat caller knows the function does not exist

  when getOneLambdaContract called with nonexistent service
    then fails with clear error that names the absent service
      sothat caller knows the service does not exist

given caller in prod environment
  when getOneLambdaContract called
    then fails fast with clear error
      sothat introspection is blocked in prod
      sothat no schema leaks to unauthorized callers

given caller in test environment
  when getOneLambdaContract called
    then fails fast with clear error
      sothat introspection is blocked outside prep
```

## usecase.2 = contract discovery for all endpoints in a service

```
given caller in prep environment
  when getAllLambdaContracts called with valid service
    then returns Record<string, LambdaEndpointSchema> for all functions in service
      sothat caller discovers all available endpoints

  when getAllLambdaContracts called with service that has no functions
    then returns empty record
      sothat caller knows the service exists but has no endpoints

  when getAllLambdaContracts called with nonexistent service
    then fails with clear error that names the absent service
      sothat caller knows the service does not exist

given caller in prod environment
  when getAllLambdaContracts called
    then fails fast with clear error
      sothat introspection is blocked in prod

given caller in test environment
  when getAllLambdaContracts called
    then fails fast with clear error
      sothat introspection is blocked outside prep
```

## usecase.3 = handler responds to introspection requests

```
given handler with input and output zod schemas
  given env.access is prep
    when introspection request received
      then returns JSON schema for input and output
      then does not invoke handler logic
        sothat introspection is fast and side-effect free

  given env.access is prod
    when introspection request received
      then fails fast with clear error
      then does not invoke handler logic
      then does not return schema
        sothat schema is not exposed in prod

  given env.access is test
    when introspection request received
      then fails fast with clear error
        sothat introspection is blocked outside prep

given handler with input and output zod schemas
  when normal request received (not introspection)
    then invokes handler logic as usual
    then returns handler response
      sothat introspection middleware does not interfere with normal operation
```

## usecase.4 = schema accuracy

```
given handler with zod input schema
  when introspection request received in prep
    then returned input JSON schema accurately represents zod schema
      sothat callers can trust the schema

given handler with zod output schema
  when introspection request received in prep
    then returned output JSON schema accurately represents zod schema
      sothat callers can trust the schema

given handler with complex zod schemas (unions, optionals, nested objects)
  when introspection request received in prep
    then returned JSON schemas preserve structure and constraints
      sothat complex contracts are accurately represented
```
