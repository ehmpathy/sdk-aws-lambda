# playtest: runtime schema introspection

## prerequisites

- node 18+
- repo cloned and dependencies installed (`npm ci`)
- keyrack unlocked for test environment (`rhx keyrack unlock --owner ehmpath --env test`)

## sandbox

- all file operations target `.temp/` (not applicable for this feature)
- this playtest uses in-memory test execution only

## edgey paths

### invalid introspect value

**cites**: not explicitly tested (implicit via schema validation)

**expected behavior**: if caller sends `{ introspect: 'invalid' }`, handler should treat as normal payload and validate against input schema (which will fail if input schema doesn't expect `introspect` field)

---

### function not found

**cites**: `local.introspection.acceptance.test.ts` case10

## pass/fail criteria

- **pass if**: all acceptance tests pass with exit 0, no snapshot changes
- **fail if**: any test fails, any snapshot differs unexpectedly, any error message is unclear

## notes

- introspection only works when `env.access === 'prep'` — this is by design to prevent schema exposure in production
- the `{ introspect: 'schema' }` payload is a special sentinel that bypasses normal handler logic
- json-schema output uses `zod-to-json-schema` for conversion
