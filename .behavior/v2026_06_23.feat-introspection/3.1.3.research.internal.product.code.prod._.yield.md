## pattern.4 = middleware chain order [REUSE]

**location**: `src/domain.operations/genLambdaEndpoint/genLambdaEndpoint.forAskEndpoint/genLambdaEndpoint.forAskEndpoint.ts:113-119`

**current behavior**:
```ts
// [4] lines 108-119
// middleware order matters:
// 1. error handlers (onError) - must be first to catch errors from all other middleware
// 2. io logger - logs input/output for debug
// 3. trail (onBefore) - injects trail context
// 4. validation (onBefore) - validates event against schema
const middlewares = [
  genConstraintErrorMiddleware(),
  genInternalServiceErrorMiddleware(),
  genIoLoggerMiddleware({ logTranslate: config.logTranslate }),
  genTrailMiddleware(),
  genZodEventValidationMiddleware({ schema: config.schema.input }),
];
```

**relation to wish**: introspection middleware must run after trail (for log context) but before validation (introspection payload won't match input schema)

**action**: [REUSE] insert introspection middleware between trail and validation
