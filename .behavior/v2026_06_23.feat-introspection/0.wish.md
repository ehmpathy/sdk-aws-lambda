wish =

we want to fulfill the introspection dream

where each endpoint exposes the schema of its input and output

via zod schemas

and that schema is then queryable to whoever calls the endpoint with '--introspect'

but only when env.access = prep

---

that way, whoever calls the lambda can easily see the input & output contract of the endpoint, and can easily then create their own sdk to safely and reliably call that endpoint

hopefully, its as simple as to

1. add the ability for definers to slap a zod schema to declare the output of the endpoint 
2. add the native middleware that intercepts calls with `{ introspect: 'schema' }` and returns the zod schema json, that should declare it end to end

---

lets ensure we have clear and thorough acceptance and integration tests on this behavior
